The problem

Complex sessions drift when source and authority get blurred.

In long Claude sessions, a draft can start acting like a spec. A hypothesis can become a fact. A Claude suggestion can begin to sound like an operator decision. Shaping Frame keeps those crossings visible.

Hypothesis becomes fact

A possible explanation starts behaving like confirmed ground.

Draft becomes spec

Working language turns into a rule before it is accepted.

Claude becomes OP

A generated proposal starts behaving like your decision.

Tracking model

Every claim keeps its source.

Shaping Frame labels the working material in a session so Claude can distinguish authority, evidence, proposals, assumptions, and documented state.

OPOperator decision

What you decided or corrected. Highest authority inside the session.

EXTExternal material

Imported material. Evidence, not automatic approval.

AIClaude proposal

Useful output from Claude, but not accepted by default.

MPModel prior

General assumption or pattern, not project fact.

DSKDocumentary state

What the available material currently supports.

Behavior

It does not remember more. It separates better.

Shaping Frame does not add persistent memory or a background monitor. It works inside the active conversation, helping Claude preserve distinctions that usually collapse during fast iteration.

Marks origin

Keeps track of whether something came from you, Claude, external material, or the current documentary state.

Flags promotion

Makes visible when a weak claim starts being reused as if it were stronger.

Protects OP authority

Keeps Claude proposals separate from actual operator decisions.

Supports review

Makes it easier to challenge assumptions without losing the working thread.

Install

Install the skill, then anchor the instruction.

Use the skill file for Claude Desktop and the custom instruction as the operating anchor for the frame.

Skill file

Download shaping-frame.skill from the main branch and upload it in Claude Desktop.

+ → Skills → Manage Skills
Customize → Skills → +
Create skill → Upload a skill

Custom instruction

Copy CUSTOM_INSTRUCTION.txt into Claude's custom instructions.

Initials → Settings → Instructions for Claude

Boundaries

A frame, not a memory system.

Shaping Frame does not run as a background hook before every Claude response. It does not eliminate drift. It reduces drift by making source, authority, and threshold crossings explicit inside the working context.

Known limit

In long or fast sessions, some crossings may still be missed. That is a host constraint, not a skill defect.
