Shaping Frame

A cognitive layer for Claude that keeps hypotheses, decisions, imported material, Claude-generated proposals, and model assumptions from collapsing into one another.

The problem is silent promotion.

In long shaping sessions, an unapproved phrase or idea can become a premise simply because it gets reused. Shaping Frame makes that crossing visible before it hardens.

Hypothesis becomes fact

A possible explanation starts acting like confirmed ground.

Draft becomes spec

Working language turns into a rule before the operator accepts it.

Claude becomes OP

A generated proposal starts behaving like the operator's decision.

Install

1

In a Claude Desktop chat, open + -> Skills -> Manage Skills.

2

In Customize > Skills, click the + button in the center column.

3

Download shaping-frame.skill from main, then choose + Create skill -> Upload a skill and upload the downloaded file.

Activate

Open CUSTOM_INSTRUCTION.txt from main, copy its contents, and add them to Instructions for Claude.

initials -> Settings -> Instructions for Claude

Do not stop in the public skills directory. This package is a custom skill file.

For the full source and operating notes, use the README on main.

It is a frame, not a memory system.

Claude does not run this skill through a background hook before each write or decision. The frame is reconstructed through conversation context and the custom instruction anchor.

Known limit

In long or fast sessions, some threshold crossings may be missed. That is a host constraint, not a skill defect.

The skill reduces drift by making important crossings visible. It does not eliminate drift.
